## 2) Quy tắc: viết gì vào đâu?
- **OS/triết lý** → `01_OS/`
- **Context chung** → `02_GENERALS_CONTEXT/`
- **Context dự án** → `08_PROJECT_CONTEXT/`
- **Log phát sinh trong ngày** (ý tưởng/decision/spike) → `04_LOGS/` (giữ dài hạn)
- **Plan tuần/tháng/quý** → `03_PLANNING/`
- **Daily Plan** → ưu tiên chạy trên TickTick; file daily chỉ là “nhật ký hỗ trợ” (tuỳ chọn)
- **Review** (tổng kết & điều chỉnh) → `07_REVIEWS/` (giữ dài hạn)
- **Nhật ký tháng (narrative)** → `06_MONTHS/`

---

## 3) Cadence tối thiểu (ít thao tác nhất)
### Hàng ngày (10’ sáng + 10’ tối)
- Sáng: đọc Week Plan + chọn 2 Deep blocks + 3–5 Builder tasks (chạy trên TickTick).
- Tối: cập nhật Daily Review ngắn + ghi log (Decision/Idea/Spike nếu có).

### Hàng tuần (60’)
- Tạo/Update Week Plan: Big/Small/KTLO + capacity + office hours + risks + owners.

### Hàng tháng (45–60’)
- Monthly review siêu nhẹ: 3 wins / 3 lessons / 3 bottlenecks + điều chỉnh capacity.

## 4) Naming convention khuyến nghị (để agent tìm file mới nhất nhanh)
- Week Plan: `YYYY-Www_WeekPlan.md`
- Month Plan: `YYYY-MM_MonthPlan.md`
- Quarter Plan: `YYYY-Qx_QuarterPlan.md`
- Daily Plan (tuỳ chọn): `YYYY-MM-DD_DailyPlan.md`
- Daily Review: `YYYY-MM-DD_DailyReview.md`

---

## 5) Rule lưu Daily: HOT 15 ngày + Archive lạnh (đã chốt)
### 5.1 Daily HOT (chỉ giữ 15 ngày gần nhất)
- Planning Daily (tuỳ chọn): `03_PLANNING/04_DAILY/`
- Review Daily: `07_REVIEWS/Daily/`

**Quy tắc:** folder Daily chỉ giữ **15 file gần nhất** để luôn gọn.

### 5.2 Archive COLD (lưu dài hạn nhưng không làm rối)
- Daily Plan archive (nếu cần): `03_PLANNING/ARCHIVE/Daily/YYYY/MM/`
- Daily Review archive: `07_REVIEWS/ARCHIVE/Daily/YYYY/MM/`

### 5.3 Khi tới ngày thứ 16 (quy trình 2 lựa chọn)
**Option A (nhẹ nhất – khuyên dùng):**
- Xoá file cũ sau khi “chưng cất” 3 dòng vào Monthly review / Monthly journal:
  - 1 win
  - 1 lesson
  - 1 bottleneck

**Option B (muốn lưu sâu):**
- Move file cũ sang ARCHIVE theo `YYYY/MM/`

> Lưu ý: **Logs (Decision/Idea/Spike) + Weekly/Monthly reviews phải giữ dài hạn.**  
> Daily có thể cắt vì chỉ là “nhật ký vận hành”.

---

## 6) Definition of Done cho hệ LIFE_AGENT
Hệ thống chạy mượt khi bạn luôn có:
- 1 file OS đúng
- 1 file `00_CONTEXT.md` cập nhật
- 1 Week Plan hiện tại
- Logs có ghi nhận khi phát sinh (Decision/Idea/Spike)
- Review tuần/tháng giữ dài hạn
- Daily folder chỉ giữ 15 ngày (HOT) + có archive hoặc chưng cất (COLD)
